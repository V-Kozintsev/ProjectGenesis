# Game Vision

## Core Fantasy

The player begins as an unknown adventurer in a frontier village. The world is dangerous, old, and partly forgotten. Power is earned slowly through combat, exploration, equipment, quests, and reputation.

The fantasy is not "become a god immediately." It is "start small, survive, grow, and become known."

## Target Feeling

The first 30 minutes should make the player feel:

- safe inside the village;
- curious about what lies outside;
- slightly threatened by the first enemies;
- rewarded by the first item upgrade;
- attached to visible progress;
- aware that the world can expand.

## Design Pillars

### 1. Small World, Real Progress

The first version can have one village, one field, one forest, and one cave. These locations should feel useful and connected.

### 2. Readable Combat

The player should understand why damage happened, when an enemy is dangerous, when to retreat, and how equipment changes power.

### 3. Meaningful Loot

Items should not be pure noise. Early loot should visibly affect survivability, damage, or utility.

### 4. Expandable Systems

Every major system should be designed for future content:

- more races;
- more classes;
- more enemies;
- more quests;
- more zones;
- multiplayer later.

### 5. No Fake Scale

The game should not pretend to be huge before it is playable. A polished small area is better than a giant empty map.

## Inspiration Direction

The project can learn from old MMORPG structure:

- starter village;
- class identity;
- mobs by level bands;
- rare drops;
- elite enemies;
- boss respawns;
- quest hubs;
- progression through zones.

But it should not blindly copy any single game.
